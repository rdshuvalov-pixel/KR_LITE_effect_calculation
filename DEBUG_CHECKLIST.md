# Чек-лист проверки работоспособности KeepRise Lite

Документ для быстрой проверки: работает ли инструмент корректно.

---

## 1. Требуемая структура Excel-файла

### Лист «Продажи»
| Колонка       | Тип    | Обязательна |
|---------------|--------|-------------|
| product_id    | int    | ✓           |
| recorded_on   | datetime | ✓         |
| price         | float  | ✓           |
| quantity      | int/float | ✓        |
| name_full     | str    | ✓           |

### Лист «Тестовые цены»
| Колонка       | Тип    | Обязательна |
|---------------|--------|-------------|
| product_id    | int    | ✓           |
| New_Price_Start | datetime | ✓       |
| New_Price     | float  | ✓           |
| Current_Price | float  | рекомендуемо (иначе 0) |

### Лист «Себестоимость»
| Колонка   | Тип    | Обязательна |
|-----------|--------|-------------|
| product_id | int   | ✓           |
| date      | datetime | ✓        |
| cost      | float  | ✓           |
| stock     | float  | ✓ (для pre-test и фильтра) |

**Важно:** Тестовые товары = product_id из «Тестовые цены». Контрольная группа = товары из «Продажи», не входящие в «Тестовые цены». Должно быть пересечение.

## 2. Быстрая проверка (автоматический скрипт)

```bash
python3 run_smoke_test.py
# или с указанием файла:
python3 run_smoke_test.py путь/к/файлу.xlsx
```

**Ожидаемый результат:** `[PASS]` по всем пунктам. Любой `[FAIL]` — инструмент работает некорректно.

## 3. Ручная проверка через Streamlit

1. Запустить: `streamlit run app.py`
2. Загрузить Excel с корректной структурой
3. Нажать «Применить настройки»

### Критерии успеха

| № | Проверка | Где смотреть |
|---|----------|--------------|
| 1 | Нет ошибки в консоли | Без traceback |
| 2 | «Расчет завершен успешно» | Зелёное сообщение |
| 3 | Блок «Отладка» — статус OK | Раздел «Отладка» |
| 4 | Тестовых товаров > 0 | Отладка / Общие результаты |
| 5 | Валидных недель > 0 | Отладка |
| 6 | Эффект (₽) и (%) отображаются | Ключевые показатели |
| 7 | Три колонки: Рост / Без изменений / Падение | Результаты по направлениям |
| 8 | Фильтр «Без изменений» работает | Анализ товара |
| 9 | Синхронизация товара между вкладками | Анализ товара ↔ Активация цен |
| 10 | Экспорт в Word | Вкладка «Отчет-история» |
| 11 | Скачать презентацию HTML/PDF | Кнопки скачивания |

## 4. Типичные ошибки и интерпретация

| Сообщение / Симптом | Причина | Действие |
|---------------------|---------|----------|
| `KeyError` / No sheet | Нет листа | Добавить листы «Продажи», «Тестовые цены», «Себестоимость» |
| `KeyError: 'stock'` | Нет колонки stock | Добавить stock в «Себестоимость» |
| Сводка пуста, 0 валидных недель | Все недели исключены или нет pre-test | Ослабить фильтры; проверить stock и даты |
| `NaT` / datetime error | Невалидные даты | Проверить recorded_on, date, New_Price_Start |
| 0 тестовых товаров | Нет пересечения | product_id в «Тестовые цены» должны быть в «Продажи» |

## 6. Эталонный файл

**Файл:** `etalon_check.xlsx`

Создать: `python3 create_etalon_file.py`

Содержит минимум данных для прохождения расчёта:
- 2 тестовых товара (101, 102)
- 5 контрольных (201–205)
- Продажи и себестоимость за ноябрь–декабрь 2025
- New_Price_Start: 02.12.2025

Загрузите `etalon_check.xlsx` в приложение — при корректной работе должны появиться результаты (эффект, growth/decline/unchanged, экспорты).
